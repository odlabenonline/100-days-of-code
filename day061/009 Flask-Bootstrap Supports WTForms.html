One of the main reasons why we're using Flask-Bootstrap in this project is because it has one of the most convenient methods for generating forms with WTForms.

Literally, in 1 line of code, you can create your form. It's as simple as:

{{ wtf.quick_form(form) }}

What this line of code will do is generate all the labels, inputs, buttons, styling for your form just by taking the WTForm object that was passed to the template (form).

You can simply delete the entire

element and add a line to inherit the wtf support from bootstrap and use the quick_form() to generate your form.


Run your code and see the entire form laid out for you with zero effort. Also, check out the error messages from validation!


Here's the documentation on this:

https://pythonhosted.org/Flask-Bootstrap/forms.html


Now you might wonder, why did I put you through all that hassle to learn how to create a WTForm from scratch when I knew all along that you can just use the Bootstrap quick_form()? Because everything is dandy as long as it works. This quick_form is a black box, it's magic. Which is great, but what happens if your form breaks? What if it's not doing what you expect it to? how would you debug magic?

That's why it's so important to understand how things work under the hood. Once you understand it, you can take all the shortcuts.
